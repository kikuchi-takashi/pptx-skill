# ポートフォリオ管理・リスク評価スキル

保有ポジションの一覧（CSV等）を受け取り、時価評価・リスク指標算出・集中度チェック・
リバランス案の提示までを行う。市場データは `market-data-report` スキルと同じ
`yfinance` を使うが、取得・整形のロジック（`scripts/risk_metrics.py` の
`fetch_price_history()`）は本スキル独自に持っており、`fetch_data.py` とは共有していない。
複数銘柄を1つの表に横持ちで結合する必要があるため（相関行列・ポートフォリオ全体の
日次リターン計算に横持ちが必須）、日本株/米国株混在時のタイムゾーン正規化など
`fetch_data.py` にはない結合処理が入っている。データソースの考え方（yfinance・
APIキー不要）は共通だが、コードは共通化されていない点に注意。

**免責事項**: 本スキルが算出する指標・提示するリバランス案は情報提供のみを目的とし、
投資助言ではない。投資判断は必ず自己責任で行うこと。

---

## このスキルが対象とすること／しないこと

- 対象: 保有ポジションの時価評価、リスク指標算出（ボラティリティ、VaR、シャープレシオ、
  最大ドローダウン、HHI集中度、相関行列）、リバランス案（目標配分との乖離の提示）。
- 対象外: 過去のリバランスルールの勝率検証（バックテスト）。
- 対象外: 実発注（→ `order-ticket-draft` スキル）。リバランス案は「どの銘柄を
  どれだけ増減させるべきか」の数量提示までで、実行はしない。


（`tabulate` はレポート内の相関行列をMarkdown表に整形するために使う）

## 入力: 保有ポジションCSV

ユーザーから保有内容を聞き取り、以下の形式のCSVを用意する（ユーザーが既に
CSV/証券会社の明細を持っていればそれを整形して使う）。

```csv
ticker,quantity,cost_basis
7203.T,100,2100
AAPL,10,180.5
BTC-USD,0.05,6000000
```

- `ticker`: yfinance が解決できる形式（日本株は `.T` サフィックス等）。大文字小文字は
  読み込み時に大文字へ正規化されるため、`aapl` のような表記でも問題ないが、同一銘柄を
  大文字小文字違いで複数行に分けて書かないこと（正規化後に重複行とみなされ、エラーで
  処理が中止される。合算してから1行にまとめる）。
- `quantity`: 保有数量
- `cost_basis`: 平均取得単価（通貨は ticker の建値通貨に合わせる。異なる通貨が混在する
  ポートフォリオでは、レポート内でどの通貨換算かを明記する）

入力ファイルが存在しない・空・列不足・数量や取得単価が数値でない・同一銘柄の重複行
などの入力エラーは、生のスタックトレースではなく `[エラー]` 付きの分かりやすい
メッセージとともに処理を中止する（`market-scan` スキルと同じ方針）。

## 実行フロー

### Step 1: 保有ポジションCSVを確認・準備する
ユーザーからヒアリングするか、既存のCSV/明細を上記形式に整形する。銘柄数が多い場合は
そのまま渡してよい（スクリプト側で一括処理する）。

### Step 2: リスク指標を算出する
```bash
python3 scripts/risk_metrics.py --positions ./positions.csv --period 1y \
  --out ./output/portfolio_risk_report.md
```
算出する指標の定義は `references/METRICS.md` を参照。

### Step 3: レポートを読んで検証する
生成された Markdown レポートを **Read ツールで確認する**。
チェック項目:
- 時価評価額・損益が保有数量・取得単価と整合しているか（桁違いがないか）
- HHI集中度が高い（例: 上位1銘柄が資産の40%超）場合、レポート上で明示されているか
- 相関行列で「実質的に同じ値動きをする銘柄の重複保有」が見えたら指摘しているか
- リバランス案が「投資助言」ではなく「現状の配分と、ユーザーが指定した目標配分との
  差分の提示」に留まっているか（目標配分をユーザーが与えていない場合は、
  均等配分など機械的な基準を明示した上での参考値であることを明記する）

### Step 4: 報告
リスク指標のサマリー、集中度・相関の懸念点、リバランス案（あれば）をユーザーに提示する。
「これは参考情報であり最終判断は自己責任」であることを毎回添える。
