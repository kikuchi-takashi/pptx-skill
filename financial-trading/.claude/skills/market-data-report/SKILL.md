# 市場データ分析・レポート作成スキル

価格データを取得し、指標を計算し、チャートを描き、読める形のレポートに落とす。
このスキル1本で「取得 → 分析 → 可視化 → レポート化」の一連を完結させる。

**免責事項**: 本スキルが生成する分析・レポートは情報提供のみを目的とし、投資助言では
ない。投資判断は必ず自己責任で行うこと。将来の値動きを保証するものではない。

---

## このスキルが対象とすること／しないこと

- 対象: 個別銘柄・指数・為替・暗号資産の価格取得、テクニカル指標（移動平均・RSI・
  ボリンジャーバンド等）の算出、チャート画像生成、Markdown/HTML サマリーレポート作成。
- 対象外: 過去データに対する売買ルールの検証（バックテスト）。過去の指標推移を見せる
  ことはするが、「このルールなら儲かったか」という戦略検証はこのスキルの範囲外。
- 対象外: 実発注（→ `order-ticket-draft` スキル）、保有ポジションのリスク評価
  （→ `portfolio-risk` スキル）。

---

## セットアップ

```bash
pip3 install yfinance pandas matplotlib
python3 -c "import yfinance, pandas, matplotlib; print('deps OK')"
```

- データソースは既定で [yfinance](https://github.com/ranaroussi/yfinance)（Yahoo Finance
  の非公式ラッパー、APIキー不要）。無料・登録不要だが、Yahoo 側の仕様変更や
  レート制限で失敗することがある。失敗時はエラーメッセージをそのままユーザーに見せ、
  リトライ間隔を空けるか別ティッカーで疎通確認する。
- 有料データベンダー（Bloomberg, Refinitiv 等）や証券会社の相場配信APIを使いたい場合は、
  `scripts/fetch_data.py` の `fetch_prices()` を差し替える。取得ロジックとレポート生成
  ロジックは分離してあるので、後段（指標計算・チャート・レポート）はそのまま使い回せる。

## 実行フロー

### Step 1: 対象と期間を確認する
ティッカー（例: `7203.T`, `AAPL`, `BTC-USD`, `JPY=X`）と期間・足種（日足/週足等）を
ユーザーに確認するか、リクエスト文から妥当な既定値を判断する（例: 「直近の値動きを見て」
→ 過去6ヶ月・日足）。複数銘柄の比較依頼なら全銘柄をまとめて1レポートにする。

### Step 2: データ取得
```bash
python3 scripts/fetch_data.py --tickers 7203.T,9984.T --period 6mo --interval 1d \
  --out ./output/market_data.csv
```
取得失敗（ティッカー誤り・レート制限・欠損）は握り潰さずそのままユーザーに報告する。
ティッカー単位で取得自体に失敗した場合は `fetch_data.py` が `<out>.errors.txt`
（サイドカーファイル）にも書き出しており、Step3 の `make_report.py` がこれを読み込んで
レポート本文の「データ取得上の注意」に自動反映する。欠損が多い期間もレポート上に
明記される。

### Step 3: 指標計算とチャート生成
```bash
python3 scripts/make_report.py --data ./output/market_data.csv \
  --out ./output/market_report.md --charts-dir ./output/charts
```
既定で計算する指標は `references/INDICATORS.md` を参照（移動平均線 SMA25/75、RSI14、
ボリンジャーバンド±2σ、騰落率、出来高）。ユーザーが特定の指標だけを求めていれば
それに絞ってよい。

### Step 4: レポートを読んで検証する
生成された Markdown レポートと `./output/charts/*.png` を **Read ツールで開いて確認する**。
チェック項目:
- 数値レンジがおかしくないか（桁違い、通貨単位の取り違え、分割・配当調整の反映漏れ）
- チャートの軸ラベル・凡例が読めるか、日本語銘柄名が文字化けしていないか
- 欠損期間や取得エラーがレポート本文にきちんと明記されているか
- サマリー文が投資助言的な断定（「買い時」「上昇確実」等）になっていないか——事実の
  記述（「25日線を上抜けた」等）に留め、断定的な推奨表現は避ける

### Step 5: 報告
生成物のパス（レポート・チャート）をユーザーに提示する。異常値や取得エラーがあれば
併せて報告する。
