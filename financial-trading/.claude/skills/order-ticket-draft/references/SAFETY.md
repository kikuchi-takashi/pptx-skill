# 安全方針（このスキルの拡張・修正時に必ず守ること）

このファイルは `order-ticket-draft` スキルを将来変更する人（人間・エージェント問わず）
向けの、破ってはならない制約の一覧。

## 絶対にしてはならないこと

1. **発注APIの呼び出し。** 証券会社・取引所のREST/WebSocket API、FIXプロトコル等、
   「送信すると実際に注文が成立しうる」通信を行うコードを追加しない。
2. **ブラウザ自動操作での送信ボタン押下。** Selenium/Playwright等で証券会社サイトの
   注文確定ボタンをクリックする、あるいはそれに相当する操作を自動化しない。
3. **常駐ループでの自動売買。** ポーリングして条件成立時に自動発注する等、人間の
   介在なしに繰り返し発注しうる仕組みを組み込まない。
4. **「発注完了」「送信済み」を示唆する出力。** チケット生成は常に「下書き」であり、
   完了・成立を意味する文言を出力に含めない。

## してよいこと（このスキルの範囲）

- 発注内容の整理・フォーマット化（テキスト/CSV）
- ポジションサイズ・価格妥当性など、静的なルールに基づく検証・警告
- 過去の市場データ（読み取り専用）を参照した価格の乖離チェック
- 証券会社の「注文一括インポート」機能向けCSVの生成（インポート後の確認・実行は
  必ず人間が証券会社の画面上で行う前提。CSVの自動アップロードや自動インポート実行は
  含めない）

## レビュー時のチェック

このスキルのコードを変更した際は、以下を確認する:
- `scripts/` 配下に `requests`/`http.client`/`urllib` 等でネットワークへ書き込みを
  行うコードや、ブラウザ自動操作ライブラリ（selenium, playwright等）への依存が
  追加されていないか
- 生成物（チケット）の冒頭にドライラン警告が必ず入るようになっているか
- テスト・デモ実行時にも実際の証券口座やAPIキーを参照するコードパスが存在しないか
